### 🌳 #why — АКТУАЛЬНИЙ СТАН (червень 2026, baseone-патерн + мобільний 50/50)

**Візуал (НЕ змінювали):** обертова **3D-сфера з крапок+ліній-сузір'я** на 2D-canvas (plain `<script>` перед `</body>`, рендер прямо в `.gs-why-dots canvas`). Точки по Фібоначчі, ребра між сусідами (`LINK=0.33`), «проростання» гілок (adjacency+`est[]`), сезонний градієнт `latCol()` зелень→охра, вузли-іконки (12 дерев + 1 сонце через `icon()`), самообертання, `window.__whyKick()` — поштовх при зміні слайда. Пауза через IntersectionObserver + `document.hidden`. Параметри `N=116, LINK=0.33, SPD=4`; центр `cx=mob?W*0.5:W*0.70`, `cy=mob?H*0.5:H*0.55`.

**HTML-структура (після рефактору):**
- `.gs-why-panels` → `.gs-why-dots` (сфера + `.gs-why-nav` тіки) + **4 окремі `<section class="gs-why-panel" data-i>`**, у кожній `.gs-container.gs-why-inner > .gs-why-text` (num/title/desc).

**Слайдер тексту — `initWhyPin()` (2 окремі гілки!):**
- **ДЕСКТОП (`min-width:901px`)** = **baseone-патерн** (витягнули з їхнього `app.js` `initServices`): сфера `.gs-why-dots` — `position:sticky;top:0;height:100svh;margin-bottom:-100svh` (фон, тримається весь час); кожна `.gs-why-panel` (`height:100svh`) піниться `ScrollTrigger {pin:true, pinSpacing:false, scrub:true}`, її текст згасає таймлайном (`opacity→0, blur, scale .85, y`). Наступна секція наїжджає поверх. Активний слайд/каскад/тіки — `IntersectionObserver` (`rootMargin:'-45% 0px -45% 0px'`).
- **МОБІЛЬНИЙ (`max-width:900px`)** = **пінований кадр 100svh, 50/50**: `.gs-why-dots` `position:absolute;top:4.5rem;height:calc(50svh-4.5rem)` (сфера зверху, **нижче хедера** щоб шторка не перекривала); `.gs-why-panel` `position:absolute;top:50svh;height:50svh` (всі накладені, видно лише `.is-active`). Один `ScrollTrigger` на `.gs-why-panels` з `pin + snap{snapTo:1/(N-1)} + scrub:0.25`, `onUpdate→activate(round(progress*(N-1)))`. Снап = **рівно один слайд за жест**, не залежить від сили скролу.
- `activate(i)`: тіки `.is-on`, перезапуск каскаду `.in` (`gsTextUp`), `__whyKick()`. На мобільному ще тогл `.is-active` на панелі.

**⚠️ КРИТИЧНО — sticky/pin ламається через `overflow`:** `#why` і `.gs-section-orange` мали `overflow:hidden` → це робить секцію скрол-контейнером і **вбиває `position:sticky`/pin**. Виправлено на **`overflow:clip`** (`#why{overflow:clip}` + `html,body{overflow-x:clip}`) — `clip` обрізає, але НЕ створює скрол-контейнер. **НЕ повертати hidden.**

### Відомі фікси — НЕ ВИДАЛЯТИ

1. **Обрізання букви "у" (та інших літер з хвостиком) в H1/H2 анімації**

   - **НЕ ПРИБИРАТИ** ці значення навіть якщо здається що вони зайві

### Етап 1 — пріоритетні виправлення без зміни дизайну

1. **Замінити контактні заглушки на реальні посилання**
   - У футері Instagram, Telegram і WhatsApp наразі мають `href="#"`.
   - Посилання «Політика конфіденційності» також є заглушкою.
   - Потрібно отримати від клієнта URL Instagram, Telegram username або URL, номер WhatsApp і текст політики конфіденційності.

2. **Зробити `index.html` реальною головною сторінкою**
   - Зараз `index.html` перенаправляє на `geometriya-sadu-VANILLA.html` через meta refresh і JavaScript.
   - Бажано перенести production-розмітку в `index.html`, щоб прибрати зайвий редірект і технічну назву HTML-файлу з URL.
   - Старий файл можна тимчасово залишити як редірект для сумісності або видалити після перевірки.

3. **Додати повний SEO-набір**
   - Абсолютний canonical URL.
   - Open Graph теги для превʼю в Telegram, Viber, Facebook та інших сервісах.
   - Twitter Card.
   - `favicon`, `robots.txt`, `sitemap.xml`.
   - JSON-LD schema для локального бізнесу (`LocalBusiness`) із назвою, телефоном, email, регіоном роботи та посиланнями на соціальні мережі.
   - За можливості додати schema `FAQPage` для наявного блоку FAQ.

4. **Оптимізувати завантаження зображень**
   - Для зображень нижче першого екрана додати `loading="lazy"` і `decoding="async"`.
   - Додати явні `width` і `height`, щоб зменшити зсуви контенту під час завантаження.
   - Розглянути preload фонового hero-зображення `images/Heroback.webp`, оскільки воно впливає на швидкість появи першого екрана.

5. **Покращити доступність інтерактивних елементів**
   - Картки послуг зараз відкривають форму через `onclick` на `<article>`: додати всередині справжню кнопку або посилання, доступне з клавіатури.
   - Для FAQ оновлювати `aria-expanded` і додати звʼязок із відповіддю через `aria-controls`.
   - Для модального вікна додати focus trap і повернення фокуса на кнопку, яка його відкрила.
   - Для кастомного select додати клавіатурну навігацію та оновлення `aria-selected`.

6. **Спростити workflow деплою**
   - У `.github/workflows/pages.yml` залишити production-деплой лише з `main`.
   - Прибрати стару тимчасову гілку `claude/blissful-faraday-y2CrU` із тригера workflow.

1. **Не публікувати весь репозиторій у GitHub Pages**
   - Workflow зараз завантажує артефакт із `path: .`, тому разом із сайтом можуть деплоїтися бриф, backup-файли, HTML-превʼю та великі тестові відео.
   - Варіант A: видалити непотрібні демо й backup-файли після фіналізації.

2. **Перевірити й прибрати зайві медіафайли**
